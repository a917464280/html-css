<p>旧容器属性描述:</p>
<p>1.box-pack:定义子元素的(主轴)水平对齐方式,有4个值。</p>
<pre><code>   .box{
         box-pack: start | end | center | justify;
              /*主轴对齐：左对齐（默认） | 右对齐 | 居中对齐 | 左右对齐*/
   }
</code></pre>
<p>2.box-align:定义子元素的交叉轴(垂直)对齐方式，有5个值。</p>

   }
</code></pre>
<p>3.box-direction:定义定义子元素的显示方向，共3个值。</p>
<pre><code>     .box{

      }
</code></pre>

<pre><code>       .box{

           /*排列方向：水平 | 垂直 | 行内方式排列（默认） | 块方式排列 | 继承父级的box-orient*/
       }
</code></pre>
<p>5.box-lines:定义子元素超出容器是否允许子元素换行，共有2个值。</p>
<pre><code>       .box{
           box-lines: single | multiple;
           /*允许换行：不允许（默认） | 允许*/
       }
</code></pre>
<p>旧子元素属性:</p>
<p>1.box-flex:定义是否允许当前子元素伸缩</p>
<pre><code>    .item{

        /*伸缩：&lt;一个浮点数，默认为0.0，即表示不可伸缩，大于0的值可伸缩，柔性相对&gt;*/
    }

<p>为什么要写兼容语法？
首先所有 webkit 内核的浏览器（包括移动端）都支持 flex 布局，只不过一些浏览器只支持旧语法，如：display: -webkit-box</p>
<p>Android端的微信用的是QQ浏览器的X5内核，支持旧版本的 flex 布局</p>
<p>Android
2.3 开始就支持旧版本 display:-webkit-box;
4.4 开始支持标准版本 display: flex;</p>
<p>IOS
6.1 开始支持旧版本 display:-webkit-box;
7.1 开始支持标准版本 display: flex;</p>
<p>1.首先是定义容器的 display 属性：</p>
<pre><code>.box{
    display: -webkit-box; /* 老版本语法: Safari, iOS, Android browser, older WebKit browsers. */
    display: -moz-box; /* 老版本语法: Firefox (buggy) */
    display: -ms-flexbox; /* 混合版本语法: IE 10 */
    display: -webkit-flex; /* 新版本语法: Chrome 21+ */
    display: flex; /* 新版本语法: Opera 12.1, Firefox 22+ */
}
</code></pre>
<p>注意:由于旧版语法并没有列入W3C标准，所以这里不用写 display:box ，下面的语法也是一样的。</p>
<p>这里还要注意的是，如果子元素是行内元素，在很多情况下都要使用 display:block 或 display:inline-block 把行内子元素变成块元素（例如使用 box-flex 属性），这也是旧版语法和新版语法的区别之一。</p>
